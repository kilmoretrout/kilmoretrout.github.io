popgenml.data.functions.newick_to_tree

popgenml.data.functions.newick_to_tree(string, *, min_edge_length=0, span=1, time_units=None, node_name_key=None, multiplier=1.0) → TreeSequence

Create a tree sequence representation of the specified newick string.

The tree sequence will contain a single tree, as specified by the newick. All leaf nodes will be marked as samples (tskit.NODE_IS_SAMPLE). Newick names and comments will be written to the node metadata. This can be accessed using e.g. ts.node(0).metadata["name"].

Parameters:

  • string (string) – Newick string

  • min_edge_length (float) – Replace any edge length shorter than this value by this value. Unlike newick, tskit doesn’t support zero or negative edge lengths, so setting this argument to a small value is necessary when importing trees with zero or negative lengths.

  • span (float) – The span of the tree, and therefore the sequence_length of the returned tree sequence.

  • time_units (str) – The value assigned to the time_units property of the resulting tree sequence. Default: None resulting in the time units taking the default of tskit.TIME_UNITS_UNKNOWN.

  • node_name_key (str) – The metadata key used for the node names. If None use the string "name", as in the example of accessing node metadata above. Default None.

Returns:

A tree sequence consisting of a single tree.
